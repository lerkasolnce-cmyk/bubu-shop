# bu-bu — интернет-магазин детских колясок (Одесса)

Дата: 2026-07-07. Статус: дизайн утверждён устно, ждёт ревью спеки.

## Цель

Полноценный интернет-магазин для bu-bu (Instagram: @bubu_odessa): каталог на 2000–3000 товаров,
корзина, онлайн-оплата monopay, админ-панель для управления товарами и ценами.
Структура разделов — по образцу karapuzov.com.ua (структура, не копия контента/дизайна),
визуальный стиль — собственный «мягкий детский» bu-bu.

## Стек

- **Next.js 15** (App Router, TypeScript) на **Vercel** (бесплатный тариф).
- **Supabase** (бесплатный тариф): Postgres (товары, категории, заказы), Storage (фото товаров), Auth (вход в админку).
- **Tailwind CSS** для стилей.
- **monopay** (эквайринг monobank) — создание счёта через API-route, webhook подтверждения оплаты.
- **Telegram Bot API** — уведомление о новом заказе в личку Валерии.
- Языки: **UA (по умолчанию) + RU + IT + EN**, переключатель в шапке; словари в JSON. Товарные поля: name_ua/name_ru обязательные, name_it/name_en/description_it/description_en — опциональные с фолбэком на UA (заполняются через админку/CSV).
- Валюта: цены хранятся и управляются **только в грн** (главная валюта). При локали **IT** цены отображаются в **EUR** по курсу НБУ (кэш ~1 час, фолбэк-курс в env). Заказ фиксирует курс на момент покупки (`eur_rate`, `total_eur`). Оплата monopay — всегда в грн.

## Витрина (публичная часть)

Структура по мотивам karapuzov:

1. **Шапка**: логотип bu-bu, поиск, телефон, переключатель UA/RU, избранное, корзина (кол-во + сумма). Меню категорий.
2. **Главная**: баннер-слайдер, плитка категорий, «Хиты продаж», «Акции» (товары со скидкой), полоса брендов (Anex, Cybex, Espiro…), блок преимуществ (доставка по Украине, гарантия, магазин в Одессе), футер (контакты, Instagram, доставка/оплата).
3. **Каталог** `/catalog/[category]`: серверная пагинация (24 на страницу), фильтры: бренд, тип (прогулочная, 2в1, 3в1, автокресло…), цена (диапазон), наличие; сортировка (цена, новинки, популярные).
4. **Поиск**: по названию (Postgres ilike/full-text), та же сетка карточек.
5. **Карточка товара** `/product/[slug]`: галерея фото, цена/старая цена, наличие, описание, характеристики (JSON key-value), кнопки «В корзину» и «Купить сейчас», блок «Похожие товары».
6. **Корзина**: localStorage + страница корзины (кол-во, удаление, сумма).
7. **Оформление заказа**: имя, телефон, город, отделение Нової Пошти (текстовое поле), способ оплаты: «Оплатить онлайн (monopay)» или «Оплата при получении». После заказа: запись в БД + Telegram-уведомление; при monopay — редирект на страницу оплаты, webhook меняет статус заказа на «оплачен».
8. **Статические страницы**: Доставка и оплата, О нас, Контакты.

Дизайн: тёплые пастельные тона (крем/пудрово-розовый/мятный + тёмно-коричневый текст), крупные скругления, мягкие тени, дружелюбная типографика. Полностью адаптивный (mobile-first — трафик из Instagram).

## Админ-панель `/admin` (за Supabase Auth, один аккаунт)

- **Товары**: список с поиском и пагинацией; создание/редактирование/удаление; поля: названия и описания UA/RU, slug, бренд, категория, цена, старая цена, наличие (в наличии/под заказ/нет), фото (upload в Supabase Storage, несколько), характеристики, флаги «хит»/«акция».
- **Категории**: CRUD, вложенность один уровень (категория → подкатегория).
- **Заказы**: список, статусы (новый → подтверждён → отправлен → выполнен / отменён), детали заказа.
- **Импорт CSV**: загрузка файла (колонки: название, бренд, категория, цена, ...), предпросмотр, массовое создание/обновление по slug. Экспорт CSV.

## Данные (Supabase)

- `categories`: id, slug, name_ua, name_ru, parent_id, sort.
- `products`: id, slug, name_ua, name_ru, description_ua, description_ru, brand, category_id, price, old_price, stock_status, specs jsonb, images text[], is_hit, is_sale, created_at. Индексы: category_id, brand, price, text search по name.
- `orders`: id, created_at, name, phone, city, np_office, comment, items jsonb, total, payment_method, payment_status, order_status, mono_invoice_id.
- RLS: чтение товаров/категорий — публичное; запись — только авторизованный админ; заказы — insert публичный, чтение только админ.

## Безопасность и платежи

- Токен monopay и Telegram-токен — только в env-переменных Vercel, ни в коем случае не в клиентском коде.
- Создание счёта monopay — в API-route по данным корзины, сумма пересчитывается на сервере по ценам из БД (не доверяем клиенту).
- Webhook monopay проверяет подпись (X-Sign, публичный ключ mono).

## Наполнение

Стартовый каталог: ~40–60 реальных моделей Anex / Cybex / Espiro (коляски 2в1/3в1, прогулочные, автокресла) с ориентировочными ценами в грн — Валерия правит через админку. Остальной ассортимент — через CSV-импорт.

## Что понадобится от Валерии (не блокирует старт)

1. Аккаунт Supabase (создадим вместе, бесплатно).
2. Аккаунт Vercel (бесплатно) + домен (опционально).
3. Токен эквайринга monopay (кабинет monobank → Эквайринг → API).
4. 5 минут на создание Telegram-бота (@BotFather) для уведомлений.
5. Логотип bu-bu, если есть (иначе нарисуем текстовый).

## Этапы

1. Каркас Next.js + Supabase, схема БД, seed-данные.
2. Витрина: главная, каталог, товар, корзина, checkout (заказ без оплаты) + Telegram.
3. Админка: товары, категории, заказы, CSV.
4. monopay-оплата + webhook.
5. UA/RU, SEO (метатеги, sitemap), полировка, деплой.

## Вне рамок (пока)

Онлайн-выбор отделений НП через API, отзывы, личный кабинет покупателя, сравнение товаров, блог.
